# TigerStyle 核心规则速查

## 优先级
**Safety > Performance > Developer Experience**

## 安全规则
- 禁止递归（除非有界证明）
- 条件拆分为嵌套 if/else，必须处理正反两空间
- 每个 if 分支追问：对应的 else 需要处理吗？
- 断言检测程序员错误，不是运行时错误
- 成对断言：写前 + 读后
- 结构体就地初始化（out pointer），避免中间拷贝
- 变量最小作用域，靠近使用处声明
- 循环和队列必须有固定上界
- 所有错误必须处理，禁止 `_` 丢弃错误

## 性能规则
- 设计阶段解决性能问题（架构 > 事后 profiling）
- 瓶颈优先级：网络 → 磁盘 → 内存 → CPU
- Batching 是主要优化手段
- 区分控制平面和数据平面
